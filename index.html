About

My research direction

My current work centers on using computation and AI to make semiconductor and materials research more predictive, automated, and experimentally useful. I am especially interested in connecting atomic-level mechanisms to process-level kinetics and device-level outcomes.

Rather than treating simulation as a standalone calculation, I view it as a closed-loop decision system: generate hypotheses, run physics-based models, quantify uncertainty, decide whether to advance/reject/hold a candidate, and use feedback to improve the next simulation or experiment.

Workflow

How I think about atomic-to-device simulation

The goal is not only to compute a number. The goal is to build a reliable workflow that produces an actionable scientific decision: advance, reject, hold, or run the next targeted simulation/experiment.

  1. Hypothesis: define the material/process/device question.
  2. Atomistic model: compute mechanisms, barriers, rates, or defects.
  3. Kinetics: translate mechanisms into process-level behavior.
  4. Device handoff: pass compact, validated parameters into TCAD/PDE models.
  5. Decision: quantify uncertainty and recommend the next action.
